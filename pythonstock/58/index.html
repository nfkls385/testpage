喺講個python backtest方法同code之前，有需要講講我個人backtest策略嘅流程


Backtest 一條Strategy 個人認為主要要經過以下呢幾個steps


1. Set up一個null hypothesis (H0)：例如我認為a,b,c 三個factors 能夠預測Stock Price （例如我覺得每次裂口高開果日就會升，相反就跌）


2.Formulate 個hypothesis： set up個trading logic, 例如幾時Buy幾時Sell幾時Hold（裂口高開幾多先買呢？）


3.Test the hypothesis: 通常用historical data去test, 再利用quantitative reasons 去prove / disapprove 個strategy (例如sharpe ratio - 識stat嘅人會知道sharpe ratio個formula (E[R]-Rf)/std 其實係t-statistics嘅化身，足夠大嘅sharpe ratio可以證明條strategy 係statistically significant *當然#of trades要夠大, 姐係mainly by skill but not luck)


4. 如果個hypothesis 係正確，就可以做optimisation: Optimisation通常都會有Objective同Constraint, 例如想minimise 個MDD同時要constraint number of trade>100, Python就會幫你試曬parameter 嘅所有possible values 再return 一條符合你input條件嘅一set parameters


5. 第4步之後如果你得到一條好靚嘅strategy，唔好咁開心住，因為呢個時候條strat都有機會overfitted/over-optimised/curve-fitting, 姐係個strategy only work well with historical data, 所以就要做forward testing, 姐係用模擬倉實際行個strategy say 一個月

但除左呢種曬時間嘅方法之外，forward test前都可以試下將parameter 做n+1嘅動作再睇下條equity curve 有冇大變化：

例如我optimise左條24 天sma cross 55天sma 嘅strategy, 賺200%，但我將24改做25天，55改做56天之後，條strategy 變成-100%，咁好明顯條strategy 就over optimised
